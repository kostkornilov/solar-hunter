# SolarHunter: домен + HTTPS в Yandex Cloud (шаги 5, 6, 7)

Инструкция продолжает уже начатую настройку и закрывает:
- проверку, что шаг 5 завершен корректно;
- шаг 6: Cloud CDN + привязка сертификата и доменов;
- шаг 7: финальная DNS/HTTPS проверка.

Домен в примерах: `solar-hunter.ru`.

---

## Шаг 5 (проверка): сертификат действительно выпущен

Откройте в Yandex Cloud:
`Certificate Manager` -> ваш сертификат.

### Что должно быть в карточке сертификата
- Статус: `Issued`.
- Тип: `Let's Encrypt`.
- Домены: `solar-hunter.ru` и `www.solar-hunter.ru`.
- Проверка домена: DNS challenge в статусе успеха.

Если статус еще `Pending`:
- не удаляйте `_acme-challenge` записи;
- дождитесь распространения DNS (до 24 часов, иногда дольше);
- проверьте NS и challenge-записи через публичный DNS:

```powershell
nslookup -type=ns solar-hunter.ru 8.8.8.8
nslookup -type=cname _acme-challenge.solar-hunter.ru 8.8.8.8
nslookup -type=cname _acme-challenge.www.solar-hunter.ru 8.8.8.8
```

## Шаг 6: настроить Cloud CDN для домена и HTTPS

Цель шага: домен должен смотреть не напрямую на bucket website endpoint, а на CDN-ресурс с вашим сертификатом.

### 6.1 Создать CDN-ресурс
1. В консоли Yandex Cloud откройте `Cloud CDN`.
2. Нажмите `Создать ресурс` (или `Создать CDN-ресурс`).
3. В блоке `Источник / Origin` выберите источник типа Object Storage website endpoint.
4. Введите endpoint вашего фронтенда:
   - формат: `<bucket>.website.yandexcloud.net`
5. Протокол до origin:
   - обычно `HTTP` для website endpoint.
6. Сохраните источник и перейдите далее.

### 6.2 Подключить пользовательские домены (hostnames)
1. В созданном CDN-ресурсе откройте раздел `Домены` / `Hostnames`.
2. Нажмите `Добавить домен`.
3. Добавьте:
   - `solar-hunter.ru`
   - `www.solar-hunter.ru`

### 6.3 Привязать сертификат
1. В том же CDN-ресурсе откройте раздел `HTTPS` / `Сертификат`.
2. Выберите `Сертификат из Certificate Manager`.
3. Выберите ваш сертификат со статусом `Issued`.
4. Сохраните изменения.

### 6.4 Дождаться применения конфигурации CDN
- Обычно 5-20 минут.
- В статусе ресурса должно быть что-то вроде `Active` / `Deploy complete`.

### 6.5 Получить целевой адрес CDN
После создания/обновления CDN у ресурса будет целевой CDN hostname (например, в интерфейсе `cdn endpoint`, `cname target` или аналогичное поле).

Скопируйте это значение — оно нужно для DNS-записей в шаге 7.

## Шаг 7: добавить/обновить DNS-записи и проверить работу

Ниже описано для Cloud DNS в Yandex Cloud (зона уже делегирована на NS Яндекса).

### 7.1 Открыть DNS-зону
1. Откройте `Cloud DNS`.
2. Выберите публичную зону `solar-hunter.ru`.
3. Откройте `Ресурсные записи`.

### 7.2 Записи для домена

В большинстве конфигураций CDN достаточно:
- `www` как `CNAME` на CDN hostname;
- `@` (корень) на целевое значение, которое показывает интерфейс CDN для apex-домена.

Создавайте записи строго по подсказкам Cloud CDN в карточке домена, потому что формат для корня (`@`) зависит от конкретного режима.

Минимум, который нужен почти всегда:
1. Нажмите `Создать запись`.
2. Поля:
   - `Имя`: `www`
   - `Тип`: `CNAME`
   - `TTL`: `300`
   - `Значение`: `<ваш-cdn-hostname>.`
3. Сохраните.

Для корня `solar-hunter.ru`:
1. Смотрите в Cloud CDN рекомендуемую запись для apex (`@`).
2. Создайте запись точно в предложенном формате (тип и значение не изменяйте вручную).

Важно:
- если в интерфейсе запись показана с точкой на конце FQDN, оставляйте точку;
- challenge-записи `_acme-challenge...` не удаляйте.

### 7.3 Проверка DNS-резолва

Проверяйте через публичные резолверы (не только через локальный роутер):

```powershell
nslookup solar-hunter.ru 8.8.8.8
nslookup www.solar-hunter.ru 8.8.8.8
nslookup solar-hunter.ru 77.88.8.8
nslookup www.solar-hunter.ru 77.88.8.8
```

### 7.4 Проверка HTTPS в браузере
Откройте:
- `https://solar-hunter.ru`
- `https://www.solar-hunter.ru`

Проверьте:
- страница фронтенда загружается;
- сертификат валиден и выдан на ваш домен;
- нет ошибки `NET::ERR_CERT_COMMON_NAME_INVALID`.

### 7.5 Проверка API-вызовов фронтенда
- Откройте DevTools -> Network.
- Убедитесь, что запросы идут на backend URL из `frontend/config.js`.
- Если есть CORS/смешанный контент, проверьте:
  - что backend доступен по HTTPS;
  - что в `frontend/config.js` указан корректный `https://...` backend URL.

## Быстрый чек-лист "готово к продакшену"

- Сертификат в Certificate Manager в статусе `Issued`.
- В Cloud CDN подключены оба домена: `solar-hunter.ru` и `www.solar-hunter.ru`.
- К CDN привязан правильный сертификат.
- DNS-записи зоны указывают на CDN (по подсказкам CDN).
- Оба адреса открываются по HTTPS без предупреждений.
- Фронтенд корректно обращается к backend API.
